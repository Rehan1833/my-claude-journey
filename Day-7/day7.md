Recommended Primary Model
🟣 Claude Sonnet 5
Why This Model Fits Me

Coding: Handles Python/debugging/small ML scripts well — you don't need Opus-level depth for diploma-level projects yet.
Learning support: Explains concepts clearly with good pacing — ideal for first-principles learning.
Cost-to-quality: Best balance for someone using Claude daily — Opus daily gets expensive/slow for routine work.
Creative + content work: Solid at LinkedIn posts, explanations, and structuring educational content for your 60-day challenge.


When to Use Haiku
✅ Quick syntax lookups ("what's the pandas function for X")
✅ Fast fact-checks or definitions
✅ High-volume simple Q&A while studying (flashcard-style)
❌ Don't use for anything requiring multi-step reasoning or code debugging
When to Use Sonnet
✅ Your daily driver — coding help, debugging, ML concept explanations
✅ Writing/structuring LinkedIn posts and prompt engineering content
✅ Reviewing your resume/project write-ups
✅ Most "learning support" conversations
When to Use Opus
✅ Deep research into a hard ML/DL concept you're stuck on
✅ Designing your portfolio project architecture (the 5-project GenAI roadmap)
✅ Debugging a genuinely tricky bug after Sonnet struggled
✅ Reviewing/refining a big deliverable (final resume project section, capstone plan)
❌ Skip for daily quick questions — overkill and slower

Recommended Effort Level
Low
Quick syntax checks, definitions, fast chat-style answers. Use with Haiku or Sonnet.
Standard (High — this is Claude's default)
Use this ~80% of the time. Good balance of depth and speed for coding help, explanations, and content drafts.
High → Extra
Reserve for genuinely hard reasoning: a tricky ML concept, an architecture decision, or debugging something non-obvious.
Max
Rare — save for the hardest problems: designing a full project from scratch, deep research synthesis, or when High/Extra visibly fell short.
Biggest Mistakes I Should Avoid

❌ Using Opus for everyday questions — wastes time/cost with no real quality gain for simple tasks.
❌ Staying on Low effort for coding/debugging — you'll get shallow fixes that don't hold up.
❌ Never escalating — if Sonnet's answer feels shallow on a hard ML concept, bump to Opus + High rather than re-asking the same way.
❌ Skipping documentation of projects — this is a resume gap you already know about; make sure your Claude workflow includes writing up each project, not just building it.


Final Recommendation

If you could only pick ONE model + ONE effort level:
Claude Sonnet 5 at Standard (High) effort.

Why: It covers ~85% of your daily needs — coding help, ML learning, content creation — at a strong quality level without the slowdown/cost of Opus. Save Opus for the rare high-stakes moments (deep research, project architecture, tricky bugs) rather than making it your default
